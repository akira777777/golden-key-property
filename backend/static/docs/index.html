Documentation

Golden Key — документация платформы

Технический справочник по информационному каталогу недвижимости: архитектура, эндпоинты, схема данных, развёртывание и требования комплаенса.

Быстрый старт

Golden Key — это FastAPI-приложение с SQLite и статическим фронтендом. Платформа намеренно узкая: каталог объектов и приём контактных запросов. Никаких платежей, кабинетов, KYC или инвестиционных продуктов.

Локальный запуск

bash 
git clone https://example.com/golden-key-dubai.git
cd golden-key-dubai/backend
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

Docker

bash 
docker build -t golden-key .
docker run --rm -p 8000:8000 \
  -e ALLOWED_ORIGINS=http://localhost:8000 \
  -e RATE_LIMIT_MAX=5 -e RATE_LIMIT_WINDOW=60 \
  golden-key

Переменные окружения

.env 
PROPERTY_LISTINGS_DB_PATH=property_listings.db
ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:5500
RATE_LIMIT_MAX=5
RATE_LIMIT_WINDOW=60

Обзор архитектуры

Сервис построен на FastAPI. Каждый запрос проходит через стек middleware, который выставляет защитные заголовки (CSP, X-Frame-Options, Permissions-Policy, Referrer-Policy) и применяет ограничение скорости к POST /api/inquiries. Данные хранятся в SQLite — это сделано намеренно: каталог информационный, без платежей и без требований распределённых транзакций.

  • Backend: FastAPI, SQLite, Pydantic v2.
  • Frontend: статические index.html + app.js + styles.css, без сборщика.
  • i18n: единый словарь в i18n.js для четырёх локалей (ru/en/de/es), переключение сохраняется в localStorage.
  • Security: заголовки выставляются централизованно (apply_security_headers), HTML/script-теги санитизируются на уровне Pydantic-валидаторов.
  • Rate limit: in-memory sliding window по IP, защищает только POST /api/inquiries.
Примечание: Сервис не использует очереди, фоновые задачи или внешние сервисы — каждый запрос полностью синхронный и читаемый в одном файле (backend/main.py).

Справочник по API

Публичная поверхность намеренно узкая: чтение каталога, детализация объекта, виртуальный тур, похожие предложения и приём контактного запроса. Любые ответы содержат поле notice с правовой оговоркой.

Список эндпоинтов

Метод Путь Назначение
GET /health Проверка доступности сервиса и БД.
GET /api/properties Пагинированный каталог с фильтрами.
GET /api/properties/{id} Детальная карточка объекта.
GET /api/properties/{id}/tour Метаданные виртуального тура.
GET /api/properties/{id}/similar До трёх похожих объектов (±30% по цене).
POST /api/inquiries Создание контактного запроса. Под rate-limit.
Комплаенс: В API нет и не будет эндпоинтов /api/payment, /api/kyc, /api/withdrawal или /api/dashboard — каталог остаётся информационным.

/api/properties

Возвращает пагинированный список объектов. Фильтры: location (подстрочный поиск), minPrice, maxPrice, listingStatus. По умолчанию отдаются только ACTIVE объекты.

curl 
curl -sS "https://example.com/api/properties?page=1&pageSize=10&location=dubai"
python 
import httpx

response = httpx.get(
    "https://example.com/api/properties",
    params={"page": 1, "pageSize": 10, "location": "dubai"},
)
response.raise_for_status()
catalog = response.json()
print(catalog["pagination"], len(catalog["data"]))
javascript 
const params = new URLSearchParams({ page: 1, pageSize: 10, location: "dubai" });
const res = await fetch(`/api/properties?${params}`);
const catalog = await res.json();
console.log(catalog.pagination, catalog.data.length);

/api/properties/{id}

Возвращает одну запись и оговорку. 404 NOT_FOUND, если идентификатор неизвестен.

curl 
curl -sS https://example.com/api/properties/1

/api/properties/{id}/tour

Возвращает tourType (NONE, PHOTO_360, MODEL_3D, VIDEO_3D) и URL тура. Поле isLocalModel подсказывает фронтенду, что нужно монтировать локальный .glb в <model-viewer>.

curl 
curl -sS https://example.com/api/properties/1/tour

/api/properties/{id}/similar

Подбирает до трёх объектов в коридоре ±30 % по цене, исключая исходный. Источник должен существовать.

curl 
curl -sS https://example.com/api/properties/1/similar

/api/inquiries (POST)

Создаёт контактный запрос. Тело — JSON с camelCase-полями. Поле consentToContact обязательно. Ответ содержит идентификатор запроса, но не эхо персональных данных.

curl 
curl -sS -X POST https://example.com/api/inquiries \
  -H "Content-Type: application/json" \
  -d '{
    "propertyId": 1,
    "fullName": "Alex Example",
    "email": "alex@example.com",
    "phone": "+971 50 000 0000",
    "message": "I would like to arrange a viewing next week.",
    "consentToContact": true
  }'
Внимание: POST /api/inquiries ограничен скользящим окном (RATE_LIMIT_MAX запросов в RATE_LIMIT_WINDOW секунд). Превышение вернёт 429 RATE_LIMITED.

/health

Возвращает {"status": "ok"} при доступной БД. При недоступности — 503.

curl 
curl -sS https://example.com/health

Модель данных

SQLite-схема создаётся в initialize_database при старте. Колонки images и features хранят JSON-строки, парсятся через _safe_json_list. Колонка asking_price_usd — единственная денежная величина, хранится исключительно как справочное значение.

SQL 
CREATE TABLE IF NOT EXISTS listings (
    id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    location TEXT NOT NULL,
    asking_price_usd REAL NOT NULL CHECK (asking_price_usd > 0),
    description TEXT NOT NULL,
    image_url TEXT,
    bedrooms INTEGER CHECK (bedrooms >= 0),
    bathrooms REAL CHECK (bathrooms >= 0),
    area_sq_m REAL CHECK (area_sq_m > 0),
    listing_status TEXT NOT NULL CHECK (listing_status IN ('ACTIVE', 'PENDING', 'SOLD')),
    tour_type TEXT NOT NULL DEFAULT 'NONE'
        CHECK (tour_type IN ('NONE', 'PHOTO_360', 'MODEL_3D', 'VIDEO_3D')),
    tour_url TEXT,
    images TEXT DEFAULT '[]',
    latitude REAL,
    longitude REAL,
    year_built INTEGER CHECK (year_built > 1800 AND year_built < 2100),
    parking INTEGER CHECK (parking >= 0),
    features TEXT DEFAULT '[]',
    created_at TEXT NOT NULL
);

CREATE TABLE IF NOT EXISTS inquiries (
    id INTEGER PRIMARY KEY,
    property_id INTEGER NOT NULL REFERENCES listings(id),
    full_name TEXT NOT NULL,
    email TEXT NOT NULL,
    phone TEXT,
    message TEXT NOT NULL,
    consent_to_contact INTEGER NOT NULL CHECK (consent_to_contact IN (0, 1)),
    created_at TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'RECEIVED'
        CHECK (status IN ('RECEIVED', 'CONTACTED', 'CLOSED'))
);
Примечание: В схеме намеренно нет колонок для балансов, кошельков, транзакций, реферальных кодов или статусов KYC.

Фронтенд: как это собрано

Главная страница (/) — статический HTML + небольшой vanilla-JS. Никаких сборщиков и зависимостей, кроме @google/model-viewer для 3D-тура.

i18n.js

Словарь TRANSLATIONS хранит ключи для ru/en/de/es. Каждый [data-i18n="key"] получает текст при загрузке и при клике по переключателю языка. Ключи с data-i18n-attr="..." применяются к атрибуту (например, content для meta или placeholder для инпута).

app.js

Главный клиент: загружает /api/properties, рендерит карточки, открывает диалоги (<dialog>) для формы запроса и тура. data-open-inquiry / data-close-inquiry — единая точка входа/выхода диалога.

styles.css

Единый файл стилей для всей платформы. CSS-переменные вынесены в :root: палитра, отступы, типографика. docs.css наследует переменные и не переопределяет базовые токены.

Диалоги

Используется нативный <dialog> — фокус-трап и закрытие по Esc работают «из коробки». Для тура <model-viewer> подключается через CDN; app.js лениво подгружает только при открытии.

Развёртывание

Сервис намеренно stateless поверх локальной SQLite. Это упрощает деплой на Vercel или в Docker-контейнер без миграций и очередей.

Vercel

Используйте конфигурацию vercel.json в корне репозитория: rewrite всего, что не начинается с /api/, на backend/main.py как serverless-функцию. SQLite подключается по PROPERTY_LISTINGS_DB_PATH — выберите /tmp для эфемерного состояния или подключите Vercel Blob, если нужна персистентность.

Docker

В backend/Dockerfile используется официальный python:3.11-slim. EXPOSE 8000, запуск через uvicorn main:app --host 0.0.0.0 --port 8000. Для compose-окружения можно прокинуть том на PROPERTY_LISTINGS_DB_PATH.

Переменные окружения

  • ALLOWED_ORIGINS — список origin, через запятую.
  • RATE_LIMIT_MAX — максимум запросов POST /api/inquiries на окно (по умолчанию 5).
  • RATE_LIMIT_WINDOW — длина окна в секундах (по умолчанию 60).
  • PROPERTY_LISTINGS_DB_PATH — путь к SQLite-файлу.

Эксплуатация и мониторинг

Сервис пишет структурированные логи (logger = logging.getLogger("golden_key")). Значимые события: создание запроса, подбор похожих, превышение rate limit, падение health-check.

/health

Подходит для liveness/readiness в Kubernetes, Render или Fly. Делает SELECT 1 FROM listings LIMIT 1, поэтому одновременно является smoke-тестом схемы.

Логи

Формат %(asctime)s [%(levelname)s] %(name)s: %(message)s. События поиска похожих объектов и приёма запросов идут с префиксом similar_properties и inquiry_created — удобно фильтровать по grep.

Сброс rate-limit

Лимиты живут в in-memory _rate_buckets. При перезапуске процесса окна сбрасываются автоматически. Для ручного сброса в long-running окружениях можно импортировать _rate_buckets и очистить его.

Комплаенс и оговорки

Юридическая оговорка: Listing information is illustrative and subject to agent and owner verification. It is not an offer, investment advice, or a binding contract.

Без платежей

Сервис не принимает деньги, не открывает кошельки, не выплачивает доходность, не продаёт инвестиционные доли и не проводит KYC. Все запросы остаются контактными: после заполнения формы представитель связывается с клиентом вручную.

Дубай: DLD / RERA

В Дубае любая сделка по объекту должна сопровождаться лицензированным брокером, зарегистрированным в Dubai Land Department (DLD) и RERA. Информация на сайте носит ознакомительный характер и не заменяет проверку:

  • проверьте RERA-номер брокера и его актуальный статус;
  • запросите подтверждение права собственности через DLD;
  • не подписывайте платёжные документы и формы без проверки условий у лицензированного специалиста.
Внимание: Любые расчёты «доходности», «гарантированной аренды» или «возврата инвестиций» не имеют отношения к этому сервису. Не переводите средства по реквизитам, найденным на сайте, и не сообщайте документы через форму.

Часто задаваемые вопросы

Это инвестиционная платформа?

Нет. Golden Key — информационный каталог недвижимости и сервис приёма контактных запросов. Сайт не предлагает инвестиционные продукты, не принимает средства и не гарантирует доходность.

Что произойдёт после отправки формы?

Запрос попадает в очередь консультантов. С вами связываются по выбранному объекту и только по оставленным контактам. Без согласия на обработку форма не отправится.

Можно ли платить через сайт?

Нет. Сайт не принимает платежи, не хранит реквизиты и не выставляет счета. Любые финансовые расчёты происходят за пределами платформы — под контролем вашего банка и лицензированного брокера.

Откуда берутся цены в каталоге?

Цены запрашиваются у владельца или его представителя и регулярно пересматриваются. Финальная стоимость может отличаться — её нужно подтверждать в переписке или на встрече с агентом.

Гарантируется ли наличие объекта?

Нет. Доступность зависит от владельца и рынка. Статус ACTIVE означает, что объект опубликован, но не заменяет оффер или договор.

Как проверить брокера в Дубае?

Запросите RERA-номер (Broker ID) и сверьте его через публичные реестры Dubai Land Department. Не работайте с брокером без действующей лицензии и актуальной регистрации.

Хранит ли сайт мои документы?

Нет. Через форму мы получаем только имя, контакт и текст сообщения. Не присылайте паспорт, банковские реквизиты или иные документы — мы их не обрабатываем и не запрашиваем.

Как связаться для уточнений?

Используйте форму на странице объекта или на главной — кнопка «Запросить подбор». Мы отвечаем в порядке очереди запросов.
